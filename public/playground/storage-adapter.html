SOG LOD Storage Adapter System

Stream Spatially Ordered Gaussians (SOG) LOD splats from multiple backends with secure credential management.

Supported Backends

📁 Local Upload
Upload a SOG bundle ZIP file from your computer for instant preview and testing.
☁️ Cloudinary CDN
Stream from Cloudinary with global edge caching. Supports multiple credential sources:
  • GitHub Secrets (GitHub Actions)
  • Render.com environment variables
  • Netlify environment variables

Quick Start

1. Local ZIP Upload:
Use the sidebar to upload a SOG bundle ZIP file. The adapter will extract and stream chunks.
2. Cloudinary Direct:
Enter your Cloudinary cloud name and folder path in the sidebar.
3. With Secrets:
Deploy on GitHub Actions, Render, or Netlify. Credentials are automatically discovered.

Key Features

  • 🔐 Secure credential management via platform-specific secret stores
  • ⚡ Parallel chunk loading and caching
  • 🌐 CDN support with global delivery
  • 📦 Local ZIP extraction for in-memory preview
  • 🔄 Fallback support across multiple deployment environments
  • 🎮 Ready for GPU rendering pipeline integration

Integration Pattern

The storage adapter integrates with the GPU rendering pipeline following the supersplat streaming pattern:
  • Fetch lowest LOD first (coarsest detail)
  • Decode chunked splat data
  • Dequantize packed attributes
  • Upload to GPU for rendering
  • Background load higher LODs
