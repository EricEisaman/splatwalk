# SplatWalk v0.6.2

## Highlights

- **Host Fast Nav camera-select API**: `runFastNav({ cameraSelect: { view, offsets } })`
  derives a world AABB (`regionBoundsFromCameraSelect`), pins the yellow select region /
  WASM `region_min`·`region_max`, and restores that FreeCamera view after nav instead of
  top-down `focusOnPlayer` (unless `keepCameraSelectView: false`).
- **Oval Storage Adapter** keeps that view + region through settle and nav; Church /
  Skatepark unchanged.
- **R3F** accepts the same `cameraSelect` option on `useSplatFastNavR3F` when region
  selection is used.

## Notes

- WASM **`api_version` stays `2`**. No new WASM wire fields; AABB only.
- Defaults for offsets: 10 m L/R, 15 m forward, 5 m behind, 5 m below / 15 m above.

## Upgrade

```bash
npm install @splatwalk/core@0.6.2
```

```ts
import { runFastNav, regionBoundsFromCameraSelect } from '/* host nav module */';

await runFastNav({
  viewer,
  bytes,
  cameraSelect: {
    view: {
      position: { x: -328.254, y: 7.6, z: 147.722 },
      eulerDegrees: { x: 40.5, y: -98.0, z: 0.0 },
    },
    // offsets optional — defaults above
  },
});
// When result.keptCameraSelectView is true, skip focusOnPlayer().
```
