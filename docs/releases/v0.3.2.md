# SplatWalk v0.3.2

Demo + docs release. The WASM core is unchanged; this proves and documents that
it is rendering-engine-agnostic.

## Added

- **React Three Fiber (three.js) reference integration.** A first-class R3F demo
  (route `/react`) renders real Gaussian splats and runs the full FAST NAV
  pipeline (floor field -> navmesh -> crowd -> click-to-move) on the same
  engine-agnostic WASM core and `recast-navigation` crowd as the Babylon showcase.
- **Engine-specific integration guide sections.** `docs/INTEGRATION.md` now has
  dedicated Babylon.js and React Three Fiber sections (handedness / `flip_y`
  contract, crowd + click-to-move, top-down framing) and states up front that the
  core has no dependency on any 3D engine, renderer, or UI framework.

## Changed

- **R3F demo reuses the persistent navmesh cache.** The React pipeline now shares
  the IndexedDB navmesh artifact cache used by the Babylon workbench and the
  Vuetify demo, so revisiting a splat with unchanged settings skips
  parse -> prune -> field -> Recast.

## Fixed

- **R3F demo loads splats under the production CSP.** `connect-src` now allows
  `blob:` so the GaussianSplats3D loader can fetch the splat Object URL.
- **Release workflow installs TypeScript** before assembling the package, so the
  framework-agnostic floor module compiles in CI.

## Install

```bash
npm install @splatwalk/core
```
