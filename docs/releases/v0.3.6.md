# SplatWalk v0.3.6

Deployment fix release. Restores the standalone Babylon Playground page in
production, which had regressed to a black screen with `harness: BABYLON is not
defined` because the site's Content-Security-Policy blocked its CDN scripts.

## Fixed

- **Playground broke in production with `BABYLON is not defined`.** The deployed
  nginx Content-Security-Policy in
  [`nginx.conf.template`](../../nginx.conf.template) restricted `script-src` to
  `'self'`, so the standalone [`public/playground/index.html`](../../public/playground/index.html)
  page could not load Babylon.js, the Babylon loaders, the in-browser TypeScript
  transpiler, the `@splatwalk/core` WASM module, or `recast.js` from their CDNs,
  and the harness's transpiled blob-module import was blocked as well. The
  `script-src` directive now additionally allows `blob:`,
  `https://cdn.babylonjs.com`, and `https://cdn.jsdelivr.net`; all other CSP
  directives were already permissive enough (`connect-src`, `worker-src`,
  `img-src` cover the runtime fetches, blob workers, and textures).

  This was verified Docker-free by running the actual `nginx-main.conf` +
  `nginx.conf.template` locally and loading the page in a real browser: before the
  fix the three CDN scripts were refused and `BABYLON` was undefined; after the
  fix the Gaussian-splat scene, Recast navmesh, and click-to-move agent all come
  up with zero CSP violations.
