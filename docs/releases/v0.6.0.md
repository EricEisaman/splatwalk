# SplatWalk v0.6.0

## Highlights

- **WebGPU / WebGL renderer toggle** on Babylon demos: Storage Adapter, Vuetify Fast Nav, and the homepage 3D Workbench (`Viewer.create` / `createBabylonEngine`).
- Preference **WebGPU** with **automatic WebGL fallback** when WebGPU is unavailable, init fails, or the adapter cannot host the GS work-buffer MRT.
- URL override: `?renderer=webgpu` or `?renderer=webgl`.
- Active backend chip / hint after init.
- **Download FastNav kits** on each community demo (`/integration-kits/*.zip`, built by `npm run build:kits`) plus **Download FastNav playground** (`playground.json`).
- R3F demo: matching renderer control with WebGPU disabled (Three.js splat path is WebGL-only).

## Notes

- WASM **`api_version` stays `2`**. The core remains engine-agnostic; WebGL vs WebGPU is chosen by the host Babylon app.
- WebGPU device creation uses `setMaximumLimits` so `maxColorAttachmentBytesPerSample` meets Babylon‚Äôs GS work-buffer MRT (40 bytes/sample); lower adapters fall back to WebGL.
- Published package `@splatwalk/core` keywords now include `webgpu`.
- Kits are reference glue + `INTEGRATE.md`; production contract remains `@splatwalk/core@0.6.0`.

## Upgrade

npm install @splatwalk/core@0.6.0
