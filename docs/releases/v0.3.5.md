# SplatWalk v0.3.5

Demo polish release. Makes the on-screen agent markers depth-correct across every
demo (Babylon workbench, Vuetify, R3F, and the Playground), and adds matching
`PLAYER` / `NPC` labels to the React Three Fiber demo.

## Added

- **R3F demo agent labels.** The React Three Fiber demo now renders billboard
  `PLAYER` (blue) and `NPC` (green) labels above its crowd agents, matching the
  Babylon demos. They use the same depth-correct technique as below
  ([`src/react/three/SplatNavController.ts`](../../src/react/three/SplatNavController.ts)).

## Fixed

- **Agent markers no longer pop in/out or float over splat geometry.** The agent
  marker labels in the Babylon demos (`PLAYER` / `NPC` / `SEED`, via
  `Viewer.attachMarkerLabel` in
  [`src/scene/Viewer.ts`](../../src/scene/Viewer.ts)) and the Playground demo's
  agent ring were alpha-blended, so they joined the transparent render pass and
  were sorted against the Gaussian splat (which never writes depth). That made
  them flicker in and out with the camera angle/distance and draw on top of splat
  walls and furniture.

  The root cause is the same one the player capsule already side-steps: opaque
  meshes render in the opaque pass and **write depth**, so the splat depth-tests
  against them (splats in front blend over and occlude; splats behind are
  discarded). The markers are now rendered the same way — **alpha-tested opaque**
  for the text labels (cut the glyphs out of a transparent quad), and a plain
  **opaque** ring in the Playground. Result: stable at every angle/distance and
  correctly occluded when the agent is behind splat geometry. Label textures also
  disable mipmaps so the thin text does not minify below the alpha-test cutoff and
  vanish at distance.
