# SplatWalk v0.3.4

Ingest + mutualism release. Adds antimatter15 `.splat` input, a right-handed
Babylon regression scene, and a full interactive **Babylon.js Playground** demo
(with a game-starter-style Full screen toggle), plus a single splat-ingest seam
and several loader fixes.

## Added

- **`.splat` (antimatter15) input support.** A new WASM `splat_to_ply` entry point
  (capability `splat_ingest`) decodes the fixed 32-byte `.splat` record into a
  full-fidelity 3DGS PLY. `.splat` now joins `.ply` and `.spz` everywhere in the
  app and pipeline.
- **Right-handed Babylon regression scene (Mutualism Track B / issues #3, #6).**
  The Babylon `Viewer` accepts a `rightHanded` option (`scene.useRightHandedSystem
  = true`), wired through `useBabylonViewer` and gated behind a hidden `?rh=1` URL
  flag on the Vuetify showcase (no user-facing toggle; left-handed stays the
  default). Validates that `splatwalk_oriented` output (right-handed, +Y up) lands
  in a right-handed Babylon scene with no boundary mirror; measured coincidence is
  documented in `docs/coordinate-alignment.md`.
- **"Stairs" example scene.** A `.spz` example added to all three demos (homepage
  workbench, Vuetify showcase, R3F demo), exercising the `.spz` ingest path
  end-to-end from the example menus.
- **Babylon.js Playground interactive demo (Mutualism Track A / issue #2).** A
  zero-install, copy-paste Playground snippet
  ([`public/playground/babylon-fast-nav.ts`](../../public/playground/babylon-fast-nav.ts))
  that mirrors the homepage workbench with its own in-scene UI (in the spirit of
  babylon-game-starter): it loads `@splatwalk/core` + recast.js from CDNs, renders
  a real Gaussian splat, runs `build_room_floor_mesh` (FAST NAV floor, honoring the
  `flip_y` contract), builds a Babylon Recast navmesh, spawns a crowd agent, and
  supports **click-to-move**. Includes a scene picker (`.ply` + gunzipped `.spz`),
  splat/floor/navmesh toggles, a live HUD, a top-down "focus on player" camera, a
  **Full screen** toggle (hijacks the Playground split — drops `#pg-split`'s
  fixed-pixel grid so `#canvasZone` fills the view; Fullscreen API fallback
  elsewhere; `?fullscreen=true` on load), and a **home icon link** overlay. It is
  an ES module exporting `class Playground` with a static async `CreateScene`,
  matching the Playground V2 runner's entry-export resolution.
- **Runnable Playground harness + `playground.json` export.** `/playground/`
  ([`public/playground/index.html`](../../public/playground/index.html), linked
  from the homepage nav) reproduces the Playground TS pipeline, offers a **Download
  `playground.json`** button (a byte-compatible Babylon Playground **V2 snippet**),
  and an **Open Playground ↗** button to the published snippet
  ([`#VUGYNW`](https://playground.babylonjs.com/#VUGYNW)).

## Changed

- **Single splat ingest seam: everything is normalized to PLY first.** `.spz` and
  `.splat` are converted to PLY at one boundary (`src/wasm/normalize.ts`,
  `normalizeSplatToPly`), and the Babylon viewer is fed those PLY bytes, so a
  dropped splat no longer depends on the source format's loader path.

## Fixed

- **`.spz` load no longer violates the Content Security Policy.** Normalizing `.spz`
  to PLY in WASM removes the runtime CDN decoder dependency that the app CSP blocked.
- **Removed the `.spz`-only 180°-X visual rotation hack.** Orientation is now
  consistent across `.ply` / `.spz` / `.splat`.
- **Non-PLY example scenes now load.** Example loaders derive the filename from the
  URL's real extension and ensure the core is initialized first, so `.spz` example
  scenes (e.g. "Stairs") normalize to PLY and render like dropped files.
- **Playground Full screen toggle now fills the canvas.** `#pg-split` uses
  fixed-pixel grid tracks, so hiding the editor previously left its column reserved;
  the toggle now drops the grid (`display:block`) so `#canvasZone` fills the full
  width (verified live, 597px → 1200px), with the overlay UI staying visible.
- **Playground snippet compiles under TypeScript 5.7+.** Keeps PLY bytes
  `ArrayBuffer`-backed so `new Blob([ply])` satisfies the now-generic `BlobPart`,
  without a version-specific generic annotation the Playground's TS would reject.

## Install

```bash
npm install @splatwalk/core
```
