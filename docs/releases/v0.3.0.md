# SplatWalk v0.3.0

**First publicly consumable release.** The community integration wishlist is fully
addressed, and the core is published for adoption under a permissive license.

## Install

```bash
npm install @splatwalk/core
```

MIT-licensed and free forever, including for commercial and proprietary use.

## Highlights

- **Published, scoped npm package** - the WASM core now ships as
  [`@splatwalk/core`](https://www.npmjs.com/package/@splatwalk/core): binary +
  wasm-bindgen glue + hand-authored TypeScript types + the framework-agnostic
  `floor` module, versioned in lockstep with the crate.
- **Open-core licensing (MIT, free forever)** - relicensed from AGPL-3.0 to MIT
  following the MUI X model. Everyone gets a free start; advanced capabilities are
  reserved for a future commercial `@splatwalk/core-pro` tier. The core never
  requires it. See [`LICENSING.md`](../../LICENSING.md).
- **Integration guide** - new task-oriented [`docs/INTEGRATION.md`](../INTEGRATION.md)
  covering install, preflight, the one-call room floor, the `flip_y` contract,
  Recast voxel conversion, progress, and structured failures.
- **Runnable examples** - [`examples/`](../../examples) adds a binary-only
  room-floor-to-GLB script and a Recast metres-to-voxels conversion script.

## Integrator asks addressed

| Ask | Status |
| --- | --- |
| Consumable FAST NAV preset | `fast_nav_preset()`, baked into `build_room_floor_mesh` |
| Version / capability introspection | `splatwalk_version()` / `_api_version()` / `_capabilities()` |
| Precise TypeScript types | hand-authored `.d.ts` shipped in the package |
| Structured floor-mesh failures | `RoomFloorFailure` with a stable `reason` code |
| Output handedness / up-axis / winding | `settings.output_space` |
| Recast-ready agent params | `recast_agent_defaults()` + `recast_config()` |
| Structured progress callback | `set_progress_callback()` |
| Published, versioned package | `@splatwalk/core` on npm |
| Non-AGPL embedding path | MIT core, free forever |

## Upgrade notes

- The package name changed from the unscoped working name to **`@splatwalk/core`**.
  Update imports to `@splatwalk/core` and `@splatwalk/core/floor`.
- The data contract is unchanged: every result still carries `api_version: 2`.
